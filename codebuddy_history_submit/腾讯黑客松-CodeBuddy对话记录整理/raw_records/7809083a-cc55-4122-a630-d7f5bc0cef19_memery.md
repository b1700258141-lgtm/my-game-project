## Work Context

- **Project Types**: **工作背景**
用户是游戏开发者，同时推进两个独立项目。其一为网页跑酷游戏“小马跑跑”（vanilla JS/Canvas），其二为现代万事屋题材的“万事屋炼金物语”，该项目使用 Vite + JS/TS + Phaser 3 开发，并参加了腾讯黑客松。该项目已规划并实现多个核心业务系统：成就(AchievementManager)、商店、存档、书架、炼金、委托、NPC交互、时间昼夜系统（床互动）、门位置选择（集市/郊外）。项目已搭建独立安全模块（src/security/）并接入腾讯云 CloudBase 云服务（环境 ID: wanshiwu-game-dev-d7dnulbc30e85f，地域 ap-shanghai），云函数 moderatePlayerName 同时支持 native callFunction 和 HTTP POST 两种调用方式，内置 extractPlayerName() 多格式事件体解析与 CORS OPTIONS 预检处理，可灵活对接 TMS 文本审核。

**个人背景**
用户习惯使用中文进行沟通和代码注释，偏好高度结构化的交互方式，如分点叙述、结构化表格和列表。在开发习惯上，用户要求分步骤（如 Step1 mock → Step2 真实 → Step3 修复）完成任务，并要求新功能必须在独立工作区、分支或模块中完成，严禁直接修改核心业务代码。用户强调严格的模块化原则，要求功能开关便于调试，且敏感词等配置严禁硬编码。用户偏好详细的完成报告，要求包含文件变更表以及清晰的改动前后对比。

**当前关注**
“万事屋炼金物语”正处于云服务深化接入与安全体系完善阶段。核心重心在于：1. 完善游戏安全体系，包括 nameModeration（起名校验）、behaviorGuard（行为监控）、dataValidator（数据校验）、contentSafetyProvider（内容安全抽象层）及 safeCommissionWrapper（委托安全封装），严格遵循中国大陆法规要求；2. 优化 CloudBase 云服务集成，云函数 moderatePlayerName 已实现 HTTP 端点兼容（支持 multi-format body 解析与 CORS OPTIONS），可灵活选择 native SDK 调用或 HTTP POST 方式调用 TMS 审核，严格实施 fail-closed 安全策略（审核失败必须阻止，禁止默认通过），云函数环境变量使用 TC_ 前缀（TC_SECRET_ID/KEY/REGION/TMS_ENABLED/SECURITY_CLOUD_MOCK）；3. 迭代现有核心系统。用户要求 AI 在修改文件时必须提供包含改动位置与行数的表格，并严格遵守“禁止覆盖已有文件”和“禁止写入真实密钥”的边界。每次任务请求均需提供编号列表及明确的禁止事项清单。

**近期动态**
- 搭建“万事屋炼金物语”项目基础框架，确立 Vite + Phaser 3 技术栈，并规划了包含昼夜、时间跳过、商店、委托、门位置选择在内的完整游戏循环系统。
- 构建独立安全模块（src/security/），实现起名校验、行为监控、数据校验、内容安全抽象层及委托安全封装，为后续接入云端审核奠定基础。
- 完成 CloudBase 云服务接入，云函数 moderatePlayerName 升级支持 HTTP POST 调用，集成 extractPlayerName() 多格式 body 解析与 CORS OPTIONS 预检处理，明确了“本地优先、云端兜底、失败即阻止（fail-closed）”的校验逻辑及环境变量规范。
- 确立严格的迭代与汇报规范：要求所有代码改动以表格形式列出文件名、修改内容及行数，且新功能必须在独立模块中通过最小化挂钩代码实现。
- 明确项目禁止事项清单，划定核心业务代码的保护边界，确保项目架构的长期稳定性，并在每次任务前显式声明约束。
